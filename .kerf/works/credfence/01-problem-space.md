# Credential & Spend Safety Fence — Problem Space

> Pass 1 (`problem-space`) of the `credfence` spec work. Grounded in the 2026-05-30 incident assessment (`docs/flywheel/2026-05-30-lifecycle-feasibility-and-gaps.md`) and the live code/spec tree (`specs/cognition-loop.md`, `specs/handler-pause.md`, `specs/claude-launchspec.md`, `internal/daemon/claudelaunchspec.go`, `internal/handler/claudehandler_chb006_024.go`, `cmd/harmonik/supervise/shim.go`). This is a planning artifact; it does not modify `specs/`. The 11 attached beads (`label=codename:credfence`) are its first tasks.

## 1. What is changing and why

On 2026-05-30 the user put `ANTHROPIC_API_KEY` in a repo-root `.env`, and within ~2 hours all API credit was consumed. The forensics (5 parallel investigators) found this was **not** one runaway — it was the compound failure of three independent safety holes, all of which this work closes:

1. **The leaked key reached every spawned `claude` child unscrubbed.** The daemon builds each implementer/reviewer's env via `ClaudeEnvVars` (`internal/handler/claudehandler_chb006_024.go:200`), which starts from `cfg.BaseEnv` and strips only `HARMONIK_SECRET_*` — `ANTHROPIC_API_KEY` (and `ANTHROPIC_AUTH_TOKEN`, `CLAUDE_CODE_OAUTH*`) pass straight through. `BaseEnv` traces back through `claudelaunchspec.go:336` → daemon `Config.HandlerEnv` → the daemon process env, which is seeded from `os.Environ()` at self-wrap exec (`cmd/harmonik/run.go:404`). The Pi side has the same hole twice: `cmd/harmonik/supervise/shim.go:103` and `cmd/harmonik/supervise/attach.go:58` both `syscall.Exec(resolved, command, os.Environ())`, handing the full parent environment — including the key — to whatever they launch. So claude billed pay-per-token API credits instead of running on the subscription.

2. **No spend ceiling metered the daemon's `claude` children — the literal root cause.** The one guardrail is `budget.ts` / CL-090, but it tracks **only Pi's own model turns** and defaulted `limitUsd` to `Infinity` (`.pi/extensions/flywheel/index.ts:62`, `budget.ts` constructor `?? Infinity`). The 23+ implementer + 20 reviewer `claude` sessions that actually burned the credit are invisible to it. There is no per-day-USD or max-runs cap anywhere daemon-side, and the inert Infinity default meant the one cap that did exist never fired.

3. **The `.env` holding the key was committable.** `git check-ignore .env` → exit 1 (verified live); there is no `.env` rule in `.gitignore`. A stray `git add -A` would have committed the secret to history.

**The unifying problem:** harmonik has no normative contract for **which process may hold the credential** and **what every spawn boundary owes** before launching a child, and no **single spend-governance model** that meters both the cognition layer (Pi turns) and the execution layer (daemon `claude` sessions) against one finite budget with a hard halt. Today credential handling is implicit (whatever `os.Environ()` happens to carry), and spend governance is half-built (CL-090 covers the wrong meter with an inert default). This work makes both explicit and normative.

**Why now.** This is the prime incident-prevention work and the safety prerequisite for the whole desired flywheel lifecycle — the assessment sequences it first (`credfence` before `pilot` before `reap`). Until the credential fence and unified budget exist, every other lifecycle improvement runs over a live, unmetered, unscrubbed key.

## 2. Goals

What should be true about the **system (specs)** after this work:

- **G1. A single process holds the credential.** The specs name exactly one process permitted to hold `ANTHROPIC_API_KEY` (the Pi cognition process), and forbid the daemon and every spawned `claude` child from receiving it. (hk-uiu98, hk-f2nm1)
- **G2. Every `claude` spawn boundary scrubs the key.** The specs define a normative scrub guarantee — `ANTHROPIC_API_KEY`, `ANTHROPIC_AUTH_TOKEN`, `CLAUDE_CODE_OAUTH*` MUST be absent from the env of any daemon-spawned `claude` implementer/reviewer — applied at the env-assembly boundary (`ClaudeEnvVars`) and asserted at the tmux substrate handoff. (hk-f2nm1, hk-4g32m)
- **G3. The credential is injected to Pi via a scoped source, not blanket inheritance.** The specs require Pi to receive the key via an explicit Pi-scoped env (not `os.Environ()` passthrough at `shim.go`/`attach.go`/`supervisor`), and `supervise start` to inject it from a non-committed scoped source so a fresh Pi boot authenticates without manual export. (hk-uiu98, hk-fo9zz)
- **G4. Spend governance meters BOTH layers against one budget.** The specs define a unified spend-governance model whose meter covers Pi's own turns AND the daemon-spawned `claude` implementer/reviewer sessions — a per-day-USD ceiling AND a max-runs ceiling — so the meter that fires matches the meter that spends. (hk-k3f8g)
- **G5. The budget default is finite.** The specs require a finite default day cap (e.g. 20 USD) with explicit opt-out for unlimited — no inert `Infinity`. (hk-60csa)
- **G6. Budget exhaustion hard-halts dispatch.** When the unified cap is hit, the system emits `budget_exhausted` wired into the existing `handler-pause` consumer (`handler-pause-policy-budget-exhausted-claude-code`, HP-012 `budget_scope=handler-account`) so no new `claude` sessions launch, and the cognition loop enters `budget-paused` pending operator resume. (hk-k3f8g, CL-090)
- **G7. The model tier is an operator knob, defaulting cheap.** The specs make the Pi judgment model an operator env override (`FLYWHEEL_MODEL_TIER1/2/3`), defaulting the judgment tier to Sonnet and gating Opus behind explicit opt-in; and expose a single operator-facing default for the daemon's `claude` baseline. (hk-rljho, hk-c5oxy)
- **G8. The secret is uncommittable and previewable.** `.env`/`*.env` are gitignored with a pre-commit secret scan (hk-pbs1u); and a `--dry-run`/plan-only daemon mode previews intended spawns ("N implementers + N reviewers across M beads") without launching `claude` or touching the live key (hk-cebjc).
- **G9. Retries are budgeted.** The review-loop / `no_progress` path carries a global re-dispatch/retry budget so a stuck bead cannot amplify spend through unbounded paid retries. (hk-c1ah6)

These are spec-level goals. The implementation epics (the `ClaudeEnvVars` scrub branch, the Pi-scoped env builder, the daemon spend meter, the budget-default flip, the gitignore + scan) are derived in pass-7; the success criteria below are about what the **specs describe**, per the kerf jig.

## 3. Non-goals

- **The quiet-by-default daemon / no-auto-pull (S2).** The incident's *trigger* mechanism (bare daemon auto-draining `br ready` into a fresh wave on every boot) is real but belongs to the **`pilot`** work (hk-exd7m). credfence assumes dispatch happens; it governs *credential and cost* of that dispatch, not *whether* it auto-starts.
- **Pi-driven curated dispatch and the pause/resume control plane (S5/S6/S9/S10).** Also `pilot`. credfence wires the *budget-exhaustion* hard-halt into the existing handler-pause consumer; it does NOT design the agent-callable `harmonik pause`/`resume` verb nor the `operator_pause_status` producer (that is `pilot`'s S9).
- **Crash recovery, orphan sweep, reap-on-exit, restart backoff, single-flywheel lock (lost issues #5/#8/#9).** These harden failure modes and belong to the **`reap`** work. credfence does not reap leaked live sessions or reconcile wreckage; it prevents the spend that made the wreckage expensive.
- **A new secrets-management subsystem / vault.** No keychain integration, no secret-rotation daemon, no encrypted-at-rest store. The contract is "scoped non-committed source" (e.g. a gitignored `.env` read only by `supervise start`, or an operator-exported var) — not a new infrastructure layer the user hasn't asked for.
- **Token-accurate cost accounting.** The spend meter is a *governance* mechanism (approximate USD per session + a max-runs backstop), not a billing-grade ledger. Approximation that reliably halts before runaway is sufficient; exact reconciliation against Anthropic invoices is out.
- **Changing the LLM transport substrate.** Whether Pi uses the raw Messages API or `pi-agent-core` is OQ-CL-003 / informative; credfence is normative on *who holds the key* and *what the budget covers*, not on transport.
- **CL-051 two-phase-done verification (S8).** A real gap, but it is correctness/verification, not credential or spend; the assessment files it independently and it is closer to `pilot`'s bridge work.

## 4. Constraints

- **Reuse the existing `handler-pause` budget-exhaustion consumer — do not invent a parallel halt path.** `handler-pause-policy-budget-exhausted-claude-code` already exists (`internal/daemon/composition_registry_hkndysh.go:60`) and HP-012 already specifies "pause the handler type immediately on `budget_exhausted{budget_scope=handler-account}`". The unified budget must emit into THAT consumer, not a new one. (handler-pause.md §4.x HP-012, §8 informative note)
- **Reuse the existing `ClaudeEnvVars` env-assembly boundary for the scrub.** The scrub belongs at `internal/handler/claudehandler_chb006_024.go:200` (the single point where child env is built), symmetric with the existing `HARMONIK_SECRET_*` strip — not as a scatter of `env -u` wrappers at every call site. The tmux substrate handoff (`tmuxsubstrate.go:179`, `Env: in.Env`) is the *assertion* point, not a second scrub site.
- **CL-090 is the home for the budget contract; extend it, don't fork it.** The unified spend model must be specified as an extension of `cognition-loop.md` CL-090 (per-day kill-switch) — broadening its meter from "Pi turns" to "Pi turns + daemon `claude` sessions" and flipping the default — plus, if the credential contract warrants its own spec, a new `specs/credential-isolation.md`. No third budget surface.
- **`budget_scope` field gap is a known dependency.** handler-pause.md §11 records that `budget_exhausted{handler-account}` needs a `budget_scope` field that "does not exist in control-points.md §4.5 — deferred to control-points amendment." The unified-budget event MUST carry `budget_scope=handler-account` to trip the consumer, so this work either lands or explicitly sequences that control-points amendment.
- **Backwards-compatible default behavior must be SAFER, not merely unchanged.** Unlike a feature add, the budget-default flip (Infinity → finite) and the scrub are *deliberate* behavior changes — the prior behavior was the vulnerability. The constraint is that an operator who explicitly opts into unlimited / passes the key through still can; the *default* changes to safe.
- **No credential value in any committed artifact, spec example, or event payload.** Specs, examples, and emitted events MUST reference the key by name only; the scrub/scan must not itself log the value. (CL-092 untrusted-input + secret-redaction discipline already in spirit.)
- **Mechanism-tagged, deterministic enforcement.** Per architecture.md §4.4 / handler-contract §4.4, the scrub and the spend meter are mechanism (no LLM judgment): a fixed deny-list of env keys, a deterministic USD/max-runs accounting, a deterministic event emission. The model is never consulted to decide whether to scrub or whether the budget is exhausted.

## 5. Success criteria

Concrete statements about what the specs MUST describe when this work is done:

- **SC1. Credential-holder contract.** A spec (new `specs/credential-isolation.md`, or a normative section folded into `cognition-loop.md`) states that exactly ONE process — the Pi cognition process — MAY hold `ANTHROPIC_API_KEY`, and that the daemon and every daemon-spawned `claude` child MUST NOT. The deny-list set is named normatively: `{ANTHROPIC_API_KEY, ANTHROPIC_AUTH_TOKEN, CLAUDE_CODE_OAUTH*}`.
- **SC2. Scrub guarantee at the spawn boundary.** The specs require `ClaudeEnvVars` (CHB-006 env assembly) to remove the deny-list keys from the constructed child env — symmetric with the existing `HARMONIK_SECRET_*` strip — and require the tmux substrate handoff to carry no deny-list key. The guarantee is stated as an invariant ("no daemon-spawned `claude` ever receives `ANTHROPIC_API_KEY`") with a named regression test (hk-4g32m).
- **SC3. Pi-scoped injection.** The specs require the credential to reach Pi via an explicit Pi-scoped env builder rather than `os.Environ()` passthrough; `cmd/harmonik/supervise/shim.go` + `attach.go` + the supervisor build a scoped env, and `supervise start` injects the key from a non-committed scoped source so a fresh boot authenticates (closing HANDOFF blocker #1).
- **SC4. Unified spend meter.** CL-090 (or its successor section) is rewritten so the per-day budget meters BOTH Pi turns AND daemon-spawned `claude` implementer/reviewer sessions against one cap, and adds a **max-runs** ceiling alongside the per-day-USD ceiling. The spec says how a daemon `claude` session's approximate USD is attributed to the shared meter.
- **SC5. Finite default + opt-out.** The specs define the default day cap as finite (e.g. 20 USD) and require explicit operator opt-out (`--budget-usd-per-day=unlimited` / env) for no limit. The inert `Infinity` default is removed.
- **SC6. Hard-halt wiring.** The specs describe the exhaustion path end-to-end: meter trips → `budget_exhausted{budget_scope=handler-account}` (or `flywheel_budget_exhausted`) emitted → `handler-pause-policy-budget-exhausted-claude-code` consumer pauses the `claude` handler type (HP-012) → cognition loop enters `budget-paused` → no new sessions launch → operator `harmonik supervise resume` clears it. The `budget_scope` field dependency (control-points.md §4.5) is resolved or explicitly sequenced.
- **SC7. Model-tier and baseline knobs.** The specs make the Pi judgment model an operator env override (`FLYWHEEL_MODEL_TIER1/2/3`, judgment tier defaults to Sonnet, Opus gated behind opt-in) and expose a single operator-facing default for the daemon's `claude` baseline.
- **SC8. Uncommittable secret + dry-run.** The repo `.gitignore` covers `.env`/`*.env` and a pre-commit secret scan is specified (hk-pbs1u); and a `--dry-run`/plan-only daemon mode is specified that prints intended spawns without launching `claude` or reading the live key (hk-cebjc).
- **SC9. Retry budget.** The specs add a global re-dispatch/retry budget to the review-loop / `no_progress` path so paid retries are bounded (complementing the per-day and max-runs ceilings).

## 7. Open questions for later passes

- **OQ-1 (research/design): Where does the daemon `claude` session's USD get attributed to the shared meter?** The daemon process and the Pi process are separate; the meter lives in Pi (`budget.ts`). Options: (a) daemon emits a per-run cost estimate event that Pi's meter consumes; (b) Pi estimates daemon `claude` cost from `run_started`/`run_completed` + a per-model rate table; (c) the meter moves daemon-side. Lean: (a) — daemon emits an approximate `run_cost` and Pi's unified meter sums Pi turns + run costs. Research pass to confirm the event surface.
- **OQ-2 (design): One spec or two?** Does the credential contract get its own `specs/credential-isolation.md`, or fold into `cognition-loop.md` alongside CL-090? Lean: a small dedicated `credential-isolation.md` — credential-holder discipline is a cross-cutting invariant (daemon + Pi + substrate), not purely a cognition-loop concern, and a separate spec gives the scrub regression test (hk-4g32m) a clear normative anchor.
- **OQ-3 (design): Does `max-runs` reset on the same day boundary as per-day-USD, or is it a per-session lifetime cap?** Lean: max-runs is a per-day ceiling resetting at local-midnight with the USD cap, so a single "new day" resets both; a separate per-session cap invites two reset semantics.
- **OQ-4 (design): Is the `budget_scope` field amendment in-scope here or sequenced to `pilot`/control-points?** It is a hard dependency for the hard-halt (G6/SC6). Lean: land the minimal `budget_scope` addition to control-points.md §4.5 within credfence, since the halt wiring is credfence's deliverable and cannot be specified without it.
- **OQ-5 (design): What is the non-committed scoped source for `supervise start` injection?** A gitignored `.env` read only by `supervise start` (re-using the very file that caused the incident, now safely scoped + gitignored), an operator-exported env var, or a path-configurable secrets file. Lean: gitignored `.env` (so the operator's existing muscle memory works) PLUS the `.gitignore` rule (hk-pbs1u) making it uncommittable — the fix turns the incident's vector into the sanctioned path.
- **OQ-6 (research): Approximate per-model USD rate table — where does it live and how is it kept current?** Needed for both Pi turns and daemon `claude` cost attribution. Lean: a small operator-overridable rate table in the flywheel extension config, defaulting to current published Anthropic per-MTok rates; stale rates are a governance-conservatism issue, not a correctness one.
