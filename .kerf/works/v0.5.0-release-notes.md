# Harmonik v0.5.0 — Release Notes

_Range: `v0.4.0..HEAD` (348 non-merge commits). Theme of the release: **enforcement-first quality** — the CI and review gates now actually block, and the work that had been hiding behind fail-open gates was root-fixed rather than suppressed._

## Highlights

- **Fail-closed quality gates.** The central pivot of this release: gates flipped from advisory to blocking. `check-short` no longer runs under `continue-on-error`, an APPROVE-only commit-verdict hook gates every commit, and a conformance floor now BLOCKS execution instead of merely warning. The quality machinery that already existed but had no authority now has teeth.
- **`check-short` is genuinely green.** Flipping the gate to fail-closed exposed real defects that the old fail-open gate had masked — a pasteinject `-race` data race plus ~18 isolation/saturation test flakes. All were root-fixed, not skipped.
- **Pi dual-provider switch.** Pi harness runs can now select model + provider per bead via a named-profile config map and a per-run `rc.*` override tuple (spec: `specs/pi-provider-switch.md`; epic `hk-m6uu2`). The default path is proven byte-identical to before.
- **Pi sandboxing (srt) proven end-to-end.** Pi child processes are wrapped in the `srt` Seatbelt sandbox on the real exec path, with the remote/tcp-socket and PATH/model-leak edge cases closed.
- **Captain & Crew system matured into an agent-manifest model** — typed agent folders (captain / crew / watch / admiral), a single boot skill, and a `harmonik agent check` schema verb.

## Enforcement-first quality arc

The organizing thesis (`df0672a5`): the quality machinery existed but had **no authority** — 11 suppression points let broken work merge green. The fix was to un-suppress, root-fix what the suppression was hiding, then enforce.

### Fail-closed CI + commit gates (A5)
- Dropped `continue-on-error` so `check-short` failures actually block; the A5 gate was widened to cover both srt and non-srt paths (`6d2c5d1a`, `50b470d3`).
- APPROVE-only commit gate: agent-reviewer `check-verdict` hook plus a real run script with a diff-keyed verdict cache (`6b16fd6e`, `b95f4b2c`).
- Conformance floor promoted to an **execution gate that BLOCKS** (`05670dee`), with a machine-readable per-cell verdict JSON and a registered conformance floor (`1a4a8ae4`, `06775446`).
- Tier-2 CI given a git identity and de-saturated so it runs deterministically (`8de5f74d`); hook-drift CI check added (`5e408284`).

### `check-short` green — root-fixes, not skips
- **The pasteinject data race** (`e246e895`): three package-global timing knobs (`splashDismissDelay`, `resumeSubmitRetryDelay`, `pasteVerifyBackoff`) were mutated by a test helper while production code read them under `-race`. Converted to `atomic.Int64` nanoseconds with test-only getter/setter seams — zero production behavior change, 2 DATA RACEs → 0.
- **The isolation/saturation flakes** (~18 across several commits): git default-branch `init -b main`, PL006 mtime handling, `t.Deadline`-scaled timeouts, `sync.Mutex` guards on stalewatch clocks, and dropping `t.Parallel()` on stdout-swapping help tests. Key commits: `3c146674`, `d9113529`, `096dee3a`, `56eb44d3`, `881dac09`, `1f540787`, `54bea199`.
- Root cause of the residual RED nailed down: the remaining failures were **stochastic saturation**, not a shrinking bug list — unbounded intra-package `t.Parallel` under `-race` self-saturates the 2-vCPU CI runner (oversubscription: `-parallel`=GOMAXPROCS=2 × `-p`=2 → up to 4 concurrent test goroutines on 2 vCPU), so wall-clock-deadline tests trip a different random subset each run. The convergence lever is a **`-parallel` cap** on the `check-short` `-race` recipe (caps intra-package `t.Parallel`) — proven to eliminate the moles in calibration. Note: this is *not* the `-p` package-parallelism knob, which was evaluated and rejected as it does not touch the intra-package self-saturator. `-race` stays ON; a non-gating full-parallelism `-race` job preserves real-race detection off the merge path (`364f0400`, PR #24; A5 flip = `76cca65e`, PR #27).

- WriteToMainDenied gate hardened to require a non-zero srt exit (`2e65ad96`).

### Dispatch consent-gate fix (Claude ≥2.1.204)
Claude 2.1.204/2.1.205 introduced a trust/consent modal that fires **before** an agent reaches `agent_ready` whenever a project-local `.claude/settings.json` declares a `permissions.allow` block — and it is suppressed by neither `~/.claude.json` trust keys nor `--dangerously-skip-permissions`. This silently killed daemon dispatch: agents launched, wrote no session log at all, and died at `agent_ready_timeout`. The fix was two parts, and it took the E2E gate to prove the first was incomplete:
- **#26 (`9120aa62`)** — stopped the daemon from *injecting* `permissions.allow` into materialized agent settings.
- **#29 (`e59beef8`)** — removed the `permissions.allow` block that was still *committed* in the tracked `.claude/settings.json` (a fresh worktree checkout re-introduced the modal on every dispatch). #26 was necessary but insufficient; the committed block had to go too. All load-bearing `hook-relay` hooks were preserved; developers keep local pre-approval via the gitignored `.claude/settings.local.json`.
- **GATE 0 caught it.** The mandatory pre-deploy end-to-end canary (isolated scratch daemon, real launch path) FAILED on the post-#26 build — reproducing the empty-session-log `agent_ready_timeout` and capturing the live trust modal — which is exactly what surfaced the still-committed block. Post-#29 the same canary passed with `agent_ready` in ~2.3s. A green unit suite would not have caught this; the isolated launch-path gate did.

## Upgrade notes
- **CI is now fail-closed.** `check-short` and the conformance floor block on failure — a red gate that previously merged will now stop the pipeline. Expect real failures to surface that were silently green before.
- **Commit gate.** An APPROVE-only agent-reviewer verdict hook now gates commits; commits without a passing verdict are blocked.
- **Fresh-clone setup:** run `make bootstrap` / `make install-hooks` (`9f210ff7`) to wire the hooks on a new checkout.
- **Pi provider config** is a new named-profile map — see `specs/pi-provider-switch.md`. The default path is byte-identical, so existing single-provider setups need no change.
- **Fleet default flipped to single-review** (triple-review now opt-in) (`d3acfeb3`).
- **Removed:** the dead flywheel Pi extension (auto-loaded on pi start, fork-bombed `kerf next`; flywheel retired in favor of the captain) (`353fc3c1`).
- `EV-029` (`supervisor_revival`, `model_selected`) added to the event compat table — downstream event consumers should tolerate the new types.
