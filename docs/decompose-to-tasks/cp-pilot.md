Read the discipline first; this file applies it.

**What CP is about (3-sentence summary).** CP defines harmonik's policy primitive: every Gate (transition assertion), Hook (event-matched side-effect), Guard (deterministic edge reorder), and Budget (typed allowance with dispatch-time enforcement) is a single typed `ControlPoint` record parameterized by `kind`, registered into a daemon-local registry owned by S02 (Policy Engine), and invoked by S01 (Orchestrator: Gates + Guards) or S05 (Hook System: Hooks). CP also owns the policy YAML grammar (4-section `metadata`/`roles`/`freedom_profiles` envelope + `gates[]`/`hooks[]`/`guards[]`/`budgets[]`), the `expr-lang/expr` policy-expression environment with a harmonik-level AST-step cost ceiling, and the cognition-tagged evaluator replay-safety contract (persisted `GateVerdictRecord`/`HookVerdictRecord` + 5-input canonical envelope hash + Cat-6 escalation on hash mismatch). The spec stops at each owner's boundary — it declares *what* a ControlPoint is and *what* each owner may assume, not *how* dispatch or invocation is implemented.
