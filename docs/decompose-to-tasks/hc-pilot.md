# HC Pilot — Decompose-to-Tasks Worked Example

`pilot-version: 0.1.0` — drafted 2026-04-27 against `specs/handler-contract.md` v0.3.3 (status `reviewed`) and discipline v0.8 (post EV r1 5-patch pass). Fifth instance of the decompose-to-tasks discipline applied to a normative spec; HC is the boundary-contract spec defining the Go `Handler` / `Session` / `Adapter` interfaces, the daemon-to-handler-subprocess wire protocol (NDJSON over Unix socket), the `LaunchSpec` shape, the goroutine ownership split (daemon-owned watcher + S04-owned adapter), the typed-error sentinel taxonomy (5 primary + 2 sub-sentinels), the silent-hang state machine, the secrets-redaction registry, the twin-parity invariant, and the skill-injection obligation. HC is heavily schema-and-protocol shaped: the §4 reqs are mostly single-declaration but cluster into two decision-procedure protocols (§7.1 silent-hang FSM, §7.2 launch handshake) whose constituent rules sit in §4.2 / §4.6. HC's `depends-on: [architecture, execution-model, event-model, process-lifecycle]` is the broadest of any spec drafted to date — every cross-spec edge to ar/em/ev/pl is admissible per §3.2 with no forward-deferral; only WM/CP/RC/ON cites surface as F-pilot-EV-3-style forward findings.

**What HC is about (2-3 sentence summary).** HC owns harmonik's deterministic-daemon-to-execution-shape boundary contract: the Go `Handler` / `Session` / `Adapter` interfaces; the daemon-handler wire protocol (1-MiB-cap NDJSON over `.harmonik/daemon.sock`, version-negotiated via first-message `handler_capabilities`); the `LaunchSpec` record shape; the watcher-vs-adapter goroutine split (S01 owns one watcher per session; S04 exposes one adapter per agent type); the 5-primary + 2-sub-sentinel error taxonomy with mechanism-tagged classification; the silent-hang state machine keyed off heartbeat absence (`T = 600s` MVH default + heartbeats at ≤T/2); the redaction registry middleware with compile-time payload-schema check; the twin-parity invariant (twins are subprocesses honoring the same wire); the skill-injection obligation (provision before `agent_ready`, fail-launch on unresolvable). HC depends on AR (axes, ZFC, agent_type identifier, role vocabulary, centralized-controller principle), EM (Outcome shape, Run/bead_id fields, validator surface, failure-class taxonomy and routing), EV (event envelope, payload schemas, dead-letter destination, bus consumer taxonomy), and PL (daemon socket path, daemon-as-parent-process rule, daemon-vs-orchestrator-agent boundary, orphan-reconnect retry schedule).
