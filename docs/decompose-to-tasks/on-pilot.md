**What ON is about (3-sentence summary).** ON defines harmonik's cross-cutting non-functional contract: the operator commands `harmonik pause` / `resume` / `stop --graceful|--immediate` / `upgrade [--rollback]` / `attach` / `enqueue` / `status` / `confirm-verdict` / `veto-verdict` / `list` / `inject-event` (the last via attach surface) MUST respect a between-task invariant ‚Äî pause and upgrade NEVER interrupt an in-flight run; only `stop --immediate` (and SIGKILL) abort in-flight runs ‚Äî and operator-control state transitions go through the `running ‚Üí pausing ‚Üí paused ‚Üí resuming ‚Üí running` (and `paused ‚Üí upgrading ‚Üí running`) FSM defined in ¬ß7.1 with a serializable mutex per ON-011 and a durable `.harmonik/daemon.state` marker per ON-030a. Restart from SIGTERM-or-crash MUST reach `daemon_ready` within RTO (30s p95 under the standard fixture / 300s hard ceiling) measured on a monotonic-corrected clock per ON-033, the graceful-shutdown drain executes 8 ordered steps (1: dispatcher off ‚Üí 2: in-flight runs to next checkpoint ‚Üí 3: HC handler-subprocess wait ‚Üí 3a: BI intent-log drain ‚Üí 4: EV bus fsync ‚Üí 5: memory flush ‚Üí 6: WM workspace unlock ‚Üí 7: orchestrator exit / enter `paused`) under per-step timeouts, with each step durably marked so a mid-drain crash resumes from the next-uncompleted step. ON is also the **authoritative** owner of the ¬ß8 exit-code taxonomy (24 codes ‚Äî codes 0..23 incl. code 0 success ‚Äî covering pidfile-locked, socket-bind-failed, drain-timeout-escalated, upgrade-hash-mismatch, runtime-panic, machine-ceiling-exhausted, ntm-unavailable, etc.) ‚Äî every other foundation spec consumes ON ¬ß8 by reference rather than minting its own taxonomy.
