Read the discipline first; this file applies it.

**What WM is about (3-sentence summary).** WM defines harmonik's workspace primitive: every run gets its own git worktree at canonical path `<repo>/.harmonik/worktrees/<run_id>/` with a fresh task branch `run/<run_id>` created off the current integration branch (`harmonik/integration` by default, or `harmonik/integration/<parent_bead_id_refsafe>` when bead-tied). The lease is held by the run — not by individual agents — represented as a JSON lease-lock file written atomically with fsync gates that order before `workspace_leased` event emission, and released only on terminal transitions (merged, discarded) with workspace-local durability markers preceding lock unlink. WM owns the three-level branching model, the squash-merge contract with author/committer split + scratch merge-worktree topology, the original-implementer-via-sidecar-walk conflict resolution path with 3-attempt re-dispatch cap, the session-log + metadata sidecar contract that bridges handler-written logs to CASS indexing, the failed-run persistence rule + startup orphan sweep with content-first staleness detection, the fresh-`run_id`-on-reopen-bead anti-reuse rule, and the orthogonal `interrupt_state` enum that composes with lifecycle state for in-flight workspaces only.
