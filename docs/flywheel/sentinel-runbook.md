# Flywheel Sentinel — Activation & Operations Runbook

> **Spec:** `specs/flywheel-motion.md`. **Implementation status:** `specs/flywheel-motion.md §9`.
> **Audience:** operators activating or operating the sentinel on a live harmonik project.
> **Written for:** v1 components FW1–FW6 + AC1–AC5 landed 2026-06-21.

---

## Prerequisites

The sentinel is wired into the daemon binary on `main`. You need:

1. A daemon binary built from `main` (post `e2252c3a` FW3 — includes FW1+FW2+FW3+AC1+AC2+AC3+AC5).
2. A `.harmonik/config.yaml` with a `sentinel:` block (generated by `harmonik init`; commented-out by default).
3. `harmonik` on `$PATH` (`go install` or a pre-built binary).

Check your binary is current:
```
harmonik version
# or
go version -m $(which harmonik) | grep "main module"
```

---

## Safe activation order

```
FW1+FW2 landed → add sentinel: block (observe-only)
  → deploy observe-only binary
  → watch governor_signal events against the real stream
  → validate staircase looks correct
  → flip mode: act (on a non-fleet project first)
  → validate trip/clear/all-clear blocking
  → promote to live fleet LAST
```

**Do not skip steps.** ACT mode can block dispatch in the live daemon if mis-configured. Validate in
observe mode first.

---

## Step 1 — Add the `sentinel:` block to config.yaml

`harmonik init` generates the block commented-out. To activate observe mode, uncomment and set `mode`:

```yaml
sentinel:
  mode: observe   # "observe" = emit governor_signal only, no dispatch side-effects

  # Tune these only if the defaults feel wrong after watching the signal stream.
  # window: 30m
  # warmup_window: 30m
  # sustained_windows: 2
  # liveness_no_progress_n: 10

  # movement_weights:
  #   bead_closed: 10
  #   run_completed: 10
  #   reviewer_verdict: 10
```

Restart the daemon after editing config.yaml (the daemon reads config at startup):

```bash
pkill harmonik                       # daemon self-restarts under the supervisor
# wait ~10s for the socket to rebind
harmonik queue status                # confirm the daemon is up
```

## Step 2 — Validate observe mode

Watch `governor_signal` events in the live stream:

```bash
harmonik subscribe --project . | grep governor_signal
# or tail the events file directly:
tail -f .harmonik/events/events.jsonl | grep governor_signal | python3 -m json.tool
```

Each tick that fires the governor emits a `governor_signal` JSON payload:

```json
{
  "level": "Dormant",          // or "Watching", "Active", "Halt"
  "movement_score": 45,
  "suppressed_by": "",         // non-empty when a suppression source is active
  "consecutive_zero_cycles": 0,
  "opportunity": true          // false → dormant even if movement is low
}
```

**What to look for:**
- `level` should be `Dormant` when real work is completing (bead_closed, run_completed events flowing).
- `level` moves to `Watching` then `Active` when the window goes quiet with ≥1 ready bead.
- `level` should never reach `Active` during warm-up or when there are no ready beads.
- `suppressed_by` shows the active suppression source (`"operator_attached"`, `"phase_flag"`, etc.).
- Reproduce the staircase by hand: count terminal-progress events in the last 30 minutes, multiply by weights, compare to the `movement_score`. They should match.

If the staircase looks miscalibrated, tune `movement_weights`, `window`, or `sustained_windows` and restart.

## Step 3 — Flip to ACT mode (non-fleet project first)

**Do this on a validation project first**, not the live fleet daemon.

```yaml
sentinel:
  mode: act   # enables EmitTrip / ClearTrip / G-liveness halt
```

After restarting the daemon, ACT mode adds:
- `EmitTrip` — writes a `decision_required` exception to `.harmonik/decision_acks/<token>` and emits a `decision_required` event when `ActivationActive` fires.
- `ClearTrip` — automatically clears the pending exception on the next tick where `ActivationDormant` fires (real movement detected).
- `G-liveness halt` — halts dispatch and emits `liveness_halt` after `liveness_no_progress_n` consecutive zero-movement cycles.

**Validate these behaviors before promoting to the fleet:**

1. **Trip fires:** confirm a `decision_required` event appears in `events.jsonl` after a quiet window with ≥1 ready bead past warm-up.
2. **All-clear blocked:** run `harmonik digest` — it should NOT say "nothing to do" while the exception file exists.
3. **Clear on movement:** close a bead or complete a run; confirm the next `governor_signal` shows `level: Dormant` and a `decision_acknowledged` event appears.
4. **Never clears on self-ack:** confirm that a captain reporting "all clear" without any `bead_closed`/`run_completed`/HEAD-advance does NOT clear the exception.

---

## Step 4 — Promote ACT mode to the live fleet daemon

Only after Step 3 validates cleanly:

1. Edit `.harmonik/config.yaml` on the live project, set `mode: act`.
2. Restart the fleet daemon.
3. Watch `governor_signal` events for one full `warmup_window` before concluding anything.

The fleet daemon is protected by `--protect-branch`, per-run budget, and the review loop — these are not
bypassed by the sentinel.

## Tuning reference

### Window and staircase

| Key | What it controls | When to change |
|---|---|---|
| `window` | How far back events are scored (default 30m) | Shrink if you want faster response; widen if the fleet runs in long natural lulls |
| `warmup_window` | How long after daemon start the governor is suppressed (default 30m) | Increase if the fleet always takes longer to spin up |
| `sustained_windows` | How many consecutive low-movement windows before tripping (default 2) | Increase to require longer confirmation of idleness before tripping |
| `movement_weights` | Per-event-type score contribution | Increase `bead_closed` weight if single-bead runs are the primary yardstick |

### Suppression

| Key | What it controls | When to change |
|---|---|---|
| `suppression_ttl` | How long operator-attached or dialogue-recency suppression lasts (default 10m) | Increase if the operator works in longer uninterrupted sessions |
| `attached_inactive_timeout` | How long to trust `operatorAttached` without a new event (default 5m) | Must be ≤ `suppression_ttl`; decrease if you want tighter liveness on attached-status |
| `phase_flag` + `phase_flag_expiry` | Force suppression for a declared phase (e.g. "design sprint") | Use sparingly; the expiry is mandatory; the sentinel resumes automatically on expiry |

### G-liveness

| Key | What it controls | When to change |
|---|---|---|
| `liveness_no_progress_n` | Consecutive zero-movement cycles before G-liveness halts dispatch (default 10) | Decrease on short-cycle projects; increase on projects with natural long idle stretches |

### Phase-2 done_definition (work-generates-work)

To activate the positive loop (staged deploy+verify bead generation), add a Phase-2 class:

```yaml
sentinel:
  mode: observe   # or act
  done_definition:
    harmonik-deploy: "harmonik queue status && harmonik version"
```

The value after the class name is the **verify command** — it must assert an observable post-condition.
It is run outside the work-loop after the deploy bead closes. A trivially-always-exit-0 command
(`true`, `:`, empty) is rejected at config-load with `ErrTrivialVerifyCommand`.

## Operations: clearing a stuck trip

### Automatic clear (normal path)
The pending trip clears automatically on the next daemon tick after a `bead_closed`, `run_completed`,
or HEAD-advance on `origin/main` is detected in the movement window. No operator action required.

### Manual operator clear (escape hatch)
If the fleet is genuinely blocked for a legitimate reason (ENOSPC, infra outage) and the automatic
clear cannot fire:

```bash
harmonik sentinel clear-trip [--project DIR]
```

This writes `ack_method: "operator"` to the ack-state file and appends a `decision_acknowledged` event,
unblocking dispatch. The next observation cycle re-evaluates from scratch.

**When to use:** only when the blockage is genuinely external (hardware, infra) — not as a way to dismiss
a real idleness finding. The sentinel will re-trip on the next sustained-low-movement window if the
underlying cause is not resolved.

**When NOT to use:** do not use to dismiss a trip caused by a captain that is actually idle. Fix the
idleness instead.

### Diagnosing why a trip is pending

```bash
# See the current ack-state file(s):
ls .harmonik/decision_acks/

# Inspect the most recent one:
cat .harmonik/decision_acks/<token>.json | python3 -m json.tool

# Cross-reference with events:
grep decision_required .harmonik/events/events.jsonl | tail -5 | python3 -m json.tool
```

The ack file contains `bead_ids` (the ready beads named when the trip fired) and
`has_undeployed_tail` (whether a Phase-2 tail was present). Use these to understand what the governor
saw when it tripped.

## Observing the governor live

```bash
# Real-time governor_signal events:
harmonik subscribe --project . | python3 -c "
import sys, json
for line in sys.stdin:
    try:
        ev = json.loads(line)
        if ev.get('type') == 'governor_signal':
            print(json.dumps(ev, indent=2))
    except: pass
"

# Historical governor decisions:
grep governor_signal .harmonik/events/events.jsonl | wc -l   # count of evaluations
grep '"level":"Active"' .harmonik/events/events.jsonl | tail -3 | python3 -m json.tool
```

## Promote-on-fleet checklist

Before setting `mode: act` on the live fleet daemon, confirm all of the following:

- [ ] Observe mode ran for at least one full `warmup_window` (default 30m) on the live project.
- [ ] `governor_signal` events show `level: Dormant` during normal fleet operation.
- [ ] `level: Active` was observed on a validation project when ready beads were un-dispatched.
- [ ] Trip/clear/all-clear blocking validated on the validation project (Step 3 above).
- [ ] G-liveness halt triggered correctly at `liveness_no_progress_n` cycles on the validation project.
- [ ] `harmonik sentinel clear-trip` tested and confirmed operational.
- [ ] FW4 adversary spawn (hk-jsvc) status checked: if not yet landed, the adversary must be manually spawned after a trip fires (the trip exception file is the trigger).

After confirming:
```bash
# Edit .harmonik/config.yaml: set mode: act
# Then restart:
pkill harmonik
sleep 15
harmonik queue status
```
