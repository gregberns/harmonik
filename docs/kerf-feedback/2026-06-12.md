# Kerf Beta Feedback — 2026-06-12

## BLOCKER (RESOLVED): `kerf finalize` aborts on gitignored `.kerf/<codename>` mirror

**Command:** `kerf finalize <codename> --branch <name>`

**Observed:** `git add: exit status 1` — finalize aborts before committing spec drafts
when the target repo's `.gitignore` includes `.kerf/*` (the harmonik project pattern).

**Expected:** finalize succeeds and the spec commit lands with both the `.kerf/<codename>`
mirror and the `specs/` content.

**Repro:** `kerf finalize standard-bead-dot --branch spec/standard-bead-dot` in the
harmonik project (2026-06-11). `.kerf/*` is gitignored; `git add .kerf/standard-bead-dot`
fails with exit status 1.

**Root cause:** `finalize.go` called `git add <repoSpecPath>` without `-f`; `git add`
refuses to stage gitignored paths without the force flag.

**Fix:** Changed `git add <repoSpecPath>` to `git add -f <repoSpecPath>` in
`cmd/finalize.go`. The `-f` flag stages the path regardless of gitignore rules, which
is the correct behavior for the `.kerf/` mirror (the work artifacts are meant to be
committed on the finalize branch even if the project gitignores `.kerf/` in day-to-day use).

**Status:** Fixed in kerf commit `6fe78df` (main). Kerf binary rebuilt.
Regression test added: `TestE2E_SpecFirstFinalize_GitignoresKerf`.
