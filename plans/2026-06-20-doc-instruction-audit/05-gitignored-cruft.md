## (d) Cleanup-script outline (rm targets — DO NOT auto-run destructive parts without review)

```
harmonik-cleanup.sh  (all paths relative to repo root; NEVER cd into a worktree)

TIER 1 — safe-always (idempotent):
  rm -f  daemon.test hooksystem.test harmonik harmonik-twin-claude
  rm -f  edited.env.txt          # + emit WARNING: rotate ANTHROPIC_API_KEY
  rm -f  .claude-pid .DS_Store
  rm -rf .claire/
  git worktree prune             # drop 15 missing-on-disk registrations

TIER 2 — stale backups (age-gated, default >7d):
  rm -rf .beads.bak.*/
  rm -rf .harmonik/beads-intents.bak/
  find .beads/.br_recovery -type f -mtime +7 -delete

TIER 3 — retention/prune (age- or count-gated, REVIEW before enabling):
  # br history archive — keep last 7 days
  find .beads/.br_history-archive -type f -mtime +7 -delete
  # run-context — keep last 7 days
  find .harmonik/run-context -mindepth 1 -maxdepth 1 -type d -mtime +7 -exec rm -rf {} +
  # keeper temp leftovers
  find .harmonik/keeper -name '*.tmp.*' -mtime +1 -delete
  # events.jsonl — rotate if >50MB (gzip, truncate) — NOT plain rm

NEVER TOUCH:
  .harmonik/worktrees/*          # LIVE daemon runs (cross-check git worktree list)
  .claude/worktrees/*-locked     # locked = in-flight agent
  .beads/beads.db* .beads/issues.* coverage.baseline workflow.dot
```

**Guard:** TIER-3 deletes must cross-check `git worktree list` and skip any dir backing a LIVE worktree, and must not run while the daemon is mid-merge.
