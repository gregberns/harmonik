## What the commit is

−539 net lines. Replaces the manual `restart-now` **marker → watcher-poll →
`RunOnDemand` nonce/journal/freshness-gate** state machine with a direct,
synchronous in-process path (`internal/keeper/restartnow.go`):

1. resolve the tmux pane,
2. verify the session id is a trusted primary (lowercase UUIDv4) via
   `ReadCtxFile` (which overlays the authoritative `.sid`),
3. ONE handoff-freshness check (present + mtime within a 10-min window),
4. inject `[KEEPER ACK <nonce>] received restart`, then `/clear`, then
   `/session-resume <handoff>` — all through one injectable seam.

Adds `keeper ping` (ack-only liveness), makes all keeper subcommands flag-only
(`--agent` required, positional → exit 2), routes explicit `--warn-pct/--act-pct`
into the existing `min(abs, ceil*window)` band, and pins the launch scripts to
`--warn-abs-tokens 200000 --act-abs-tokens 215000`.

## Judgment call 4(a) — pct-vs-band

**Pinning 200k/215k is acceptable to merge now; grafting the clamp is a
follow-up, not a blocker.**

- The commit does two independent things here. (i) It makes `--warn-pct/--act-pct`
  *honored* by routing them into the existing `min(abs, pctCeil*window)` seam —
  so an explicit pct can now fire *earlier* than the abs cap (good; fixes Bug 1).
  (ii) It changes the launch scripts to pass `--warn-abs-tokens 200000
  --act-abs-tokens 215000` instead of `--warn-pct 30 --act-pct 35`.
- The old script values (30/35 on a 1M window = 300k/350k) were **wider** than
  the locked 200k/215k band — naively honoring them would have *regressed* the
  operator's band-retune HARD-NO (memory: "No keeper-band retune"). Pinning to
  abs 200k/215k keeps the current, locked band exactly. So pinning is the
  **conservative, no-regression** choice and is safe to land as-is.
- The killed agent's `derivePctThreshold` "clamp pct to tighten-only" idea is a
  *nicety* (lets pct stay meaningful without ever widening). It is **not needed
  for correctness** because the merged `min(abs, ...)` already guarantees pct can
  only ever fire *earlier* than abs — a too-high pct is harmlessly capped by abs.
  The only thing the clamp adds is symmetry/ergonomics for an operator who *wants*
  to express the band in pct. **Follow-up, file a bead; not a merge-blocker.**

Caveat worth surfacing to the operator (informational, not blocking): if the
operator's actual intent was a **300k cap** (per the MEMORY "300k cap via Sonnet
watchdog" note), neither this commit nor the old script delivers that — the
commit pins the *narrower* 200k/215k. That is a config-intent decision for the
operator, independent of whether this code merges.

## Judgment call 4(b) — identity / `.sid`

**`.sid` is ALREADY the authoritative identity source, and `restart-now` now
consumes it — but this commit did not *make* it authoritative; it inherits it.**

- `ReadCtxFile` (gauge.go:49-61, hk-8prq) already overlays the SessionStart-hook
  `.sid` over the gauge's `session_id` when `.sid` is present and a primary
  UUIDv4. **This commit does NOT touch `gauge.go` or `sessionid.go`** (confirmed:
  empty `--stat`). What the commit *does* is route the manual path through
  `ReadCtxFile` + `IsPrimarySID`, so `restart-now` now benefits from `.sid`
  authority and refuses on a non-primary id (test: `TestRestartNow_UnverifiedSID_Refuses`).
- What it **fixes** for identity: the manual path no longer relies on the marker's
  embedded `session_id` field (removed) — it reads the live verified id at the
  moment of action.
- What it **leaves in place**: the watcher's gauge-rederivation / latch /
  UUIDv7-uppercase heuristics (the "session-id dead after /clear" antipattern
  workarounds) still exist for the *fallback* path when `.sid` is absent/malformed.
  This commit does not collapse identity to a single source globally — that is the
  "collapse identity to one authoritative source" item the architecture critique
  lists under Option L, and remains a separate, larger follow-up.

## Judgment call 5 — does it touch the automatic ACT-loop?

**No — manual path only. Confirmed.** `MaybeRun` (cycle.go:513) and `runCycle`
(cycle.go:683) — the automatic warn/act cycle — are fully intact; only
`RunOnDemand` + its two helpers were removed. The watcher's automatic warn/act
state machines are unchanged save for deleting the now-dead restart-now marker
detection block. **Consequence the operator must understand:** the worst observed
live failure — the *automatic* ACT-when-idle loop + handoff-truncate-to-0 (Bug 3)
— is **outside this commit's blast radius**. Landing `89852bb3` will NOT fix that
symptom. It is good, real, reversible housekeeping that makes the *manual* path
correct and loud; it is not the cure for the automatic-loop pain.
